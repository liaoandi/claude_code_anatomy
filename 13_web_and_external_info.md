# 第 13 章 Web 与外部信息获取

## Agent 怎么获取外部信息

写代码经常需要查资料：看文档、找示例、确认某个 API 的参数。Claude Code 有两个内建工具处理这件事：WebFetchTool（拉取指定 URL 的内容）和 WebSearchTool（先搜索再拉取）。

这两个工具是内建的，受同一套权限和消息机制管控，不是让 agent 自己写 curl 命令去联网。

## 为什么要有 prompt 参数

调用 WebFetchTool 必须同时提供 URL 和 prompt——"你想从这个页面里得到什么"。

这个设计有点反直觉：直接拿页面内容不就好了，为什么还要说明目的？

原因是上下文成本。一个完整的文档页面可能有几万 token，但你实际需要的可能只是其中一个函数的参数说明。有了 prompt，工具在返回内容之前会先做一轮过滤，只返回和 prompt 相关的部分，而不是把整个页面塞进上下文。

这个代价是一次额外的处理，好处是每次 web fetch 不会把上下文预算的一大块都用掉。

## 域名级权限控制

和所有其他工具一样，WebFetchTool 在执行前也过权限系统。但和文件路径、shell 命令不同，Web 权限是按域名控制的。

某些域名可以配置为预授权，直接放行，不弹确认框。某些域名需要每次确认。某些域名直接拒绝。

被拦截时，系统会给出快速规则建议，一键添加白名单，不用自己手写规则。

## WebFetchTool 不处理认证

这个工具不会帮你登录、不会携带 session cookie、不会处理 OAuth。遇到需要认证的 URL，它会直接失败，并且提示你应该用专门的 MCP tool 来处理。

这是一个清晰的边界划分：WebFetchTool 适合抓公开的网页内容；需要认证的外部系统，应该通过 MCP Server 接入，认证逻辑放在 MCP Server 侧，不是让 Claude Code 来管。
